# Algorithm Study (개인 저장용)

> 알고리즘 문제 풀이를 위한 개인 학습 정리 노트

---

## 1. 자료구조

자료구조는 문제 해결을 위한 도구로, 알고리즘 구현 시 간접적으로 사용됨.

| 자료구조 | 활용 |
|---------|------|
| **스택** | DFS, SCC 등 |
| **큐** | BFS |
| **우선순위 큐** | 다익스트라, 프림, 그리디 |
| **그래프** | 인접 리스트, 인접 행렬로 표현 |

---

## 2. 그리디 / 구현

### 그리디 알고리즘

- 매 순간 최적의 선택을 통해 전체 문제를 해결.
- 정렬과 우선순위 큐를 자주 활용.
- 예) 특정 기간 내 최대 이익 구하기: 정렬 순서 변경이나 우선순위 조정을 통해 해결.

### 구현 (시뮬레이션)

- 문제에서 제시된 조건을 바탕으로 자신이 생각한 논리를 코드로 구현.
- 알고리즘의 논리를 정확히 구현할 수 있도록 연습 필요.

---

## 3. 완전 탐색 — DFS, BFS, 백트래킹

### BFS

- 최단 거리 또는 최단 경로를 구할 때 적합.
- 같은 레벨의 노드부터 탐색하므로 최적 경로를 빠르게 찾음.

> **Tip**: 최단 거리 배열 초기 값을 `-1`로 설정하면 `visited` 배열 없이도 구현 가능.

### DFS

- 모든 경로를 탐색하거나 특정 조건을 만족하는 경로를 찾을 때 유용.
- 스택이나 재귀를 사용하여 구현. (개인적으로 재귀가 편리)
- 재귀는 함수 리턴 값을 활용해 정보를 이전 노드에 전달 가능.

### 백트래킹

- DFS에서 조건이 맞지 않으면 해당 경로를 **즉시 종료**(가지치기)하고 다음 경로로 이동.

### 비트마스크

- 완전 탐색 문제에서 효율적으로 상태를 관리.

---

## 4. 이분 탐색

### 기본 구현

- `while (left < right)` 또는 `while (left <= right)` 패턴 사용.
- 개인적으로 `while (left < right)` 선호 (코드 일관성).
- 타겟이 범위 안에 있도록 초기 범위 설정이 중요.

### 매개변수 탐색 (Parametric Search)

- 최적화 문제를 결정 문제로 변환하여, 답의 범위를 이분 탐색으로 좁혀가는 기법.
- `lower_bound`: 찾고자 하는 값의 **첫 등장 위치** 반환.
- `upper_bound`: 찾고자 하는 값보다 **큰 값이 처음 등장하는 위치** 반환.

---

## 5. 순열 / 조합

- **순열**: 방문 여부를 체크하며 구현.
- **조합**: 현재 방문한 노드의 인덱스를 다음 호출에 전달하여 구현.
- 부분 집합을 활용해 요소 수가 `r`개인 조합을 구할 수도 있음.

---

## 6. 부분집합

- 포화 이진 트리를 활용해 쉽게 이해 가능.
- 비트마스크를 활용하면 간결한 코드 작성 가능.

---

## 7. 분할정복

- 문제를 재귀적으로 작은 부분 문제로 나누어 해결 후 합침.
- **DP와의 차이점**: 부분 문제 간 중복이 없음.
- 예) 사분면 문제에서 `size = 2^d`를 활용해 사분면 판별 가능.

---

## 8. DP (동적 계획법)

핵심: 중복되는 부분 문제의 결과를 저장하여 효율적으로 해결.

### 유형

| 유형 | 설명 | 예시 |
|------|------|------|
| **점화식 기반** | 이전 상태에서 현재 상태를 유도 | 배낭 문제 |
| **탐색 기반** | 중복되는 부분 상태를 저장 | 외판원 순회, 내리막길 |

> 최소/최대 비용만 구하려면 완전 탐색으로 풀어도 되지만, 중복 계산이 많아지면 DP로 최적화.

### 접근 방식

| 방식 | 설명 |
|------|------|
| **Top-Down** | 재귀 + 메모이제이션 (필요한 부분 문제만 계산) |
| **Bottom-Up** | 반복문 + 테이블링(Tabulation) (작은 문제부터 테이블에 채워나감) |

---

## 9. 문자열 처리

### 스택 활용

- 특정 문자열을 찾아 삭제하는 문제에 유용.

### KMP

- 문자열 매칭 시 불일치가 발생하면, 실패 함수(failure function)를 통해 패턴의 어느 인덱스부터 다시 비교할지 결정.

### Trie

- 문자열 검색과 관련된 문제에서 자주 사용.

### Aho-Corasick

- KMP와 Trie를 결합해 다중 문자열 검색에 활용.

### Suffix Array

- 문자열의 모든 접미사를 정렬하여 배열로 관리.
- **카사이(Kasai) 알고리즘**으로 LCP(최장 공통 접두사) 계산.
- **Prefix Doubling**: 접두사 길이를 2배씩 늘려가며 정렬하는 기법. Suffix Array 구현 시 자주 사용.

## 10. 그래프

### 위상 정렬

- **Kahn 알고리즘**: 진입 차수(In-Degree)를 기준으로 순서를 결정 (시작점을 찾는다).
- **DFS 기반**: 후위 순회(Post-Order)의 역순을 활용.

### SCC (강한 연결 요소)

- **타잔 알고리즘**을 활용해 강한 연결 요소를 구성.
- 현재 방문한 정점이 그 서브그래프 내에서 도달할 수 있는 가장 작은 방문 순서의 정점일 때 SCC를 구성.
- `sccId`(SCC가 구성되는 순서)는 값이 클수록 위상 정렬에서 앞에 위치 (역순 관계).
- 2-SAT 문제 해결에도 활용.

### LCA (최소 공통 조상)

- DP를 통해 각 노드의 조상 관계(희소 배열)를 미리 저장.
- 공통 조상을 효율적으로 탐색.

### 단절점 / 단절선

- DFS에서 현재 정점의 방문 순서(`disc[u]`)와 자식 서브트리의 최소 방문 순서(`low[child]`)를 비교.
  - **단절점**: `disc[u] ≤ low[child]` (단, 루트는 자식 수로 판단)
  - **단절선**: `disc[u] < low[child]`

### 최단 거리

| 알고리즘 | 특징 |
|---------|------|
| **다익스트라** | 가중치 비음수 |
| **벨만-포드** | 음수 가중치 가능 |
| **플로이드-워셜** | 모든 정점 쌍 간 최단 거리 |

> [참고 블로그](https://jaeun-choi98.github.io/2023/10/05/data_structure(4))

### MST (최소 신장 트리)

- **프림**: 우선순위 큐(Heap) 활용.
- **크루스칼**: Union-Find로 구현.

### 이분 매칭

- DFS를 통해 매칭 확장.
- 현재 정점(u)에서 연결된 정점(v)을 탐색하고, v가 이미 매칭된 정점(u')이 있다면 u'가 다른 정점과 매칭될 수 있는지 재귀적으로 탐색. 가능하면 현재 정점(u)을 포함시켜 더 큰 매칭을 구성.

### 최대 유량 / 최소 컷 (Max Flow / Min Cut)

- **최대 유량 = 최소 컷** (Max-Flow Min-Cut Theorem).
- 유량의 대칭성: `flow(u→v) = -flow(v→u)`.
- 여유 유량 `cap(u→v) - flow(u→v) > 0`인 간선을 따라 증가 경로를 탐색.
- 모든 증가 경로의 유량 합 = 최대 유량.

## 11. 기하학

### CCW (Counter Clock Wise)

- 세 점의 방향(반시계, 시계, 일직선) 판별.

### 컨벡스 홀 (Convex Hull)

- 점들의 집합에서 모든 점을 포함하는 최소 볼록 다각형 생성.
- 기준점을 기준으로 점들을 반시계 방향으로 정렬 후, 볼록(시계 방향)인지 검사.

---

## 12. 기타

### 세그먼트 트리

- 구간 연산을 효율적으로 처리.
- 리프 노드에 데이터를 저장하고, 조상 노드에 연산 결과를 저장.

### 누적합

- 구간 합 계산을 최적화.
- 데이터 수정이 잦은 경우 세그먼트 트리를 활용.

### 희소 배열 (Sparse Table)

- 정적 데이터에서 구간 쿼리(최솟값, 최댓값 등)를 전처리 후 O(1)에 처리.
- LCA 문제에서 조상 관계를 저장하는 데에도 활용.

### 유니온-파인드

- 서로소 집합 관리 알고리즘.
- MST(최소 신장 트리) 구현 시 사용 (예: 크루스칼 알고리즘).
